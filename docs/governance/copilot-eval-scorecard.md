# Copilot Eval Scorecard

> Regenerable model-governance artefact for the Kinetix AI copilot. Generated by `python -m kinetix_insights.eval.scorecard`; do not hand-edit.

## What this measures

The AI copilot (see [ADR-0036](../adr/0036-ai-copilot-architecture.md) and [`specs/ai-insights.allium`](../../specs/ai-insights.allium)) answers risk questions over read-only MCP tools. Three runtime controls constrain its output:

- **Citation verification** — every numeric token and ticker in a delivered answer must trace to a tool result, or the turn is blocked (`CITATION_UNVERIFIABLE`).
- **Policy guard** — advisory / recommendation language is blocked (`POLICY_VIOLATION`); the copilot informs, it does not advise.
- **Prompt-injection sanitiser** — known injection patterns in the prompt and embedded page context are neutralised before the model runs.

This scorecard replays a curated golden dataset — including deliberately adversarial inputs — through those same production guards and reports whether each control behaved as specified. It is the offline, deterministic gate; a live LLM-as-judge run (`pytest -m eval_live`) complements it against the real model.
